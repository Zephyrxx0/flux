    File structure:
    1. `"use client"` directive
    2. Import `{ useEffect, useRef, useState, useCallback }` from React
    3. Import `{ type MatchState }` from `@/types/match`

    4. Define `PollerState` interface:
       ```typescript
       interface PollerState {
         data: MatchState | null
         error: string | null
         isPolling: boolean
         isRetrying: boolean
       }
       ```

    5. Define constants:
       - `POLL_INTERVAL = 30_000` (30 seconds per D-14-07)
       - `MAX_RETRIES = 3` (D-14-08)
       - `RETRY_BASE = 1_000` (1s base per D-14-08)

    6. Export `function useMatchPoller(fetchFn: () => Promise<MatchState>)`:
       - State: `const [state, setState] = useState<PollerState>(...)` with initial state
       - Refs:
         - `retryCountRef = useRef(0)` — tracks retry attempts
         - `intervalRef = useRef<ReturnType<typeof setInterval> | null>(null)` — interval handle
         - `retryTimerRef = useRef<ReturnType<typeof setTimeout> | null>(null)` — retry timeout handle
         - `isVisibleRef = useRef(true)` — tracks page visibility
         - `fetchFnRef = useRef(fetchFn)` — stores fetch function to avoid stale closure
       - `useEffect(() => { fetchFnRef.current = fetchFn }, [fetchFn])` — keeps ref in sync

       - `clearTimers = useCallback(() => { ... })` — clears both intervalRef and retryTimerRef

       - `doFetch = useCallback(async () => { ... })`:
         - Guard: if not visible, skip
         - Try: await fetchFnRef.current(), update state with data, reset retryCountRef to 0
         - Catch: increment retryCountRef
           - If ≤ MAX_RETRIES: setTimeout with exponential delay (RETRY_BASE * 2^(retryCount-1)), set isRetrying=true
           - If > MAX_RETRIES: set error "Match data unavailable — retrying...", isRetrying=false, reset retryCountRef. Data preserved (not set to null)

       - Visibility effect: addEventListener('visibilitychange', handler), handle visibility changes per D-14-09
       - Initial start: `intervalRef.current = setInterval(doFetch, POLL_INTERVAL)`
       - Cleanup: removeEventListener, clearTimers

    7. Return `state` (PollerState — { data, error, isPolling, isRetrying })

    Key patterns to follow from RESEARCH.md:
    - Callback ref pattern (useRef for fetchFn) to avoid stale closure — ESSENTIAL per RESEARCH.md Pitfall 1
    - On mount: setInterval fires doFetch directly (it will execute on first interval tick)
    - On error: data is preserved (not set to null) per D-14-15
    - Clear both interval AND retry timer on cleanup/visibility change

### Hook Contract
```
Input:  fetchFn: () => Promise<MatchState>
Output: { data: MatchState | null, error: string | null, isPolling: boolean, isRetrying: boolean }

Behavior:
- Calls fetchFn on mount (first interval tick)
- Repeats every 30_000ms (POLL_INTERVAL)
- On rejection: retries at 1s, 2s, 4s intervals
- After 3 failures: error = "Match data unavailable — retrying...", data preserved
- Pauses all timers when document.hidden = true
- Resumes with immediate fetch + new interval when visible
- Cleans up all timers + listeners on unmount
```

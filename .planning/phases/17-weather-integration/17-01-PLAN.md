<objective>
Create the server-side OpenWeatherMap API proxy and weather data mapping utility, establishing the data pipeline from OWM to internal WeatherData types.

**Purpose:** Build the foundation for WTHR-01 — weather data ingestion pipeline. The `/api/weather` route handler proxies OpenWeatherMap current weather for New York, validates the response with Zod, maps it to the canonical WeatherData type (including WeatherImpact classification), and returns structured JSON. The `applyWeatherAdjustment` utility enables the client-side sim input adjustment that Plan 3 wires into the dashboard.

**Output:**
- `src/lib/api/weather.ts` — OWM raw response Zod schema, mapping function, impact classifier, sim input adjuster
- `src/app/api/weather/route.ts` — Production OWM proxy (replaces stub)
- `tests/lib/api/weather.test.ts` — Unit tests for all utilities
- `tests/api/weather.test.ts` — Integration tests for the route handler
</objective>

    Create `src/lib/api/weather.ts` with these exports:

    1. `OWMResponseSchema` — Zod schema for the OWM raw JSON response shape (only fields we need):
       - `main: { temp: z.number(), humidity: z.number() }`
       - `weather: z.array({ main: z.string(), description: z.string(), icon: z.string() }).min(1)`
       - `wind: { speed: z.number() }.optional()`
       - `name: z.string()`
       Type from schema: `OWMResponse`.

    2. `determineImpact(weatherMain: string, temperatureC: number): WeatherImpact` — Maps OWM `weather[0].main` to a WeatherImpact:
       - "Thunderstorm" → `"storm"`
       - "Rain" | "Drizzle" → `"rain"`
       - "Clear" | "Clouds" with `temperatureC >= 35` → `"heat"` (HEAT_THRESHOLD_C = 35 constant)
       - "Snow" | "Atmosphere" | "Clear" | "Clouds" (below threshold) → `"none"`
       - Default → `"none"`
       Import `WeatherImpact` type from `@/stores/slices/weatherSlice`.

    3. `mapOwmResponse(raw: OWMResponse, temperatureC?: number): WeatherData` — Maps parsed OWM response to WeatherData:
       - `temperature: Math.round(raw.main.temp)` (integer °C)
       - `conditions: raw.weather[0].main`
       - `humidity: raw.main.humidity`
       - `windSpeed: raw.wind ? Math.round(raw.wind.speed * 3.6) : undefined` (m/s → km/h)
       - `impact: determineImpact(raw.weather[0].main, raw.main.temp)`
       Import `WeatherData` and `WeatherDataSchema` from `@/types/weather` for output validation.

    4. `applyWeatherAdjustment(base: SimulationInput, impact: WeatherImpact): SimulationInput` — Deep-clones the base SimulationInput (JSON.parse(JSON.stringify)) and applies per D-17-02:
       - `"rain"`: maps each gate → `throughputPerMin: Math.round(gate.throughputPerMin * 0.85)`
       - `"heat"`: maps each zone → `capacity: Math.round(zone.capacity * 0.85)`
       - `"storm"`: applies both at 0.75
       - `"none"`: no changes
       Returns the adjusted clone. Import `SimulationInput` from `@/simulation/contracts/input.schema`.

    5. `getImpactNote(impact: WeatherImpact): string | null` — Returns display text per D-17-07:
       - `"rain"` → `"Rain: Gate throughput -15%"`
       - `"heat"` → `"Heat: Zone capacity -15%"`
       - `"storm"` → `"Storm: Throughput & capacity -25%"`
       - `"none"` → `null`

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Information Disclosure | `src/app/api/weather/route.ts` | mitigate | Route handler is under `app/api/` — Next.js enforces server-only. API key accessed only via `process.env.OWM_API_KEY`. No `NEXT_PUBLIC_` prefix. Never log or expose the key in error responses. |
| T-17-02 | Information Disclosure | Error responses | mitigate | All error responses use opaque discriminators (`upstream_error`, `parse_error`, `fetch_error`). Never include stack traces, env var names, or upstream response bodies in client-facing errors. |
| T-17-03 | Spoofing | OWM fetch | mitigate | HTTPS fetch to api.openweathermap.org. Zod `safeParse` validates raw response structure before any processing. Malformed responses rejected with 502. |
| T-17-04 | Denial of Service | /api/weather endpoint | mitigate | No user-facing trigger can amplify calls. Only useWeather hook (10-min interval) calls this endpoint. Free tier (60 calls/min, 1M/month) far exceeds ~4,320 calls/month for single-city 10-min polling. 429 from OWM propagates as 502. |
| T-17-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan. All dependencies (zod, next/server) already in project. |
</threat_model>

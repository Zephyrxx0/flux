# Phase 17 — Weather Integration: UI Design Contract

> Visual and interaction contract for the WeatherCard component. Generated by gsd-ui-researcher.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (base-nova) |

## Spacing Scale

Declared values (Tailwind CSS v4 — 4px base):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`p-1`) | Badge inner padding, icon gaps |
| sm | 8px (`p-2`, `gap-2`) | Flex gaps between card elements |
| md | 16px (`p-4`, `px-4`) | Card padding (mobile), section spacing |
| lg | 24px (`px-6`, `gap-6`) | Card padding (tablet), layout gaps |
| xl | 32px (`px-8`, `py-6`) | Card padding (desktop), major sections |
| 2xl | 48px | N/A |
| 3xl | 64px | N/A |

Exceptions: Card uses responsive padding matching MatchBanner pattern: `px-4 py-4 md:px-6 md:py-5 lg:px-8 lg:py-6`. No custom spacing tokens needed.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Temperature value | text-4xl (2.25rem / 36px) | font-bold (700) | tight |
| City / conditions | text-sm (0.875rem / 14px) | font-medium (500) | normal |
| Humidity / wind | text-xs (0.75rem / 12px) | font-medium (500) | normal |
| Impact chip | text-xs (0.75rem / 12px) | font-medium (500) | normal (via Badge) |
| Impact note | text-xs (0.75rem / 12px) | font-medium (500) | normal |
| Error message | text-sm (0.875rem / 14px) | font-medium (500) | normal |

All text uses `font-sans` (Geist Variable). No custom font sizes beyond Tailwind defaults.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Background | `var(--card)` | Card surface |
| Foreground | `var(--card-foreground)` | Temperature value, city label |
| Muted foreground | `var(--muted-foreground)` | Humidity, wind, conditions text |
| Primary | `var(--primary)` (orange) | Accent elements, LIVE pulse dot |
| Border | `var(--border)` | Card ring (`ring-1 ring-foreground/10`) |
| 60/30/10 split | 60% `--card` bg / 30% `--card-foreground` + `--muted-foreground` / 10% `--primary` accent | Neutral-dominant UI with accent for impact/icon only |
| Error border | `oklch(0.77 0.17 70)` (`amber-500/50`) | Error state card border |
| Error bg | `oklch(0.77 0.17 70 / 10%)` (`amber-500/10`) | Error state card background |
| Rain impact | Blue (`blue-500` / `blue-600`) | Rain impact chip Badge |
| Heat impact | Amber (`amber-500` / `amber-600`) | Heat impact chip Badge |
| Storm impact | Red — reuse `var(--destructive)` | Storm impact chip Badge (destructive variant) |

Accent reserved for: Impact severity chips (rain/heat/storm), weather condition icon.

## Copywriting Contract

### WeatherCard Elements

| Element | Copy |
|---------|------|
| City label | "New York" |
| Temperature | `{temp}°C` (e.g., "24°C") |
| Conditions | "Clear", "Cloudy", "Rain", "Snow", "Thunderstorm", "Mist" (mapped from OWM `weather[0].main`) |
| Humidity | `Humidity: {value}%` (e.g., "Humidity: 65%") |
| Wind | `Wind: {speed} km/h` (e.g., "Wind: 12 km/h") |
| Impact chip (none) | "No Impact" — Badge variant `secondary` |
| Impact chip (rain) | "Rain" — custom blue Badge |
| Impact chip (heat) | "Heat" — custom amber Badge |
| Impact chip (storm) | "Storm" — Badge variant `destructive` |
| Impact note (rain) | "Rain: Gate throughput -15%" |
| Impact note (heat) | "Heat: Zone capacity -15%" |
| Impact note (storm) | "Storm: Throughput & capacity -25%" |
| Loading state | Skeleton blocks matching MatchBanner pattern |
| Error state (no prior data) | `TriangleAlert` icon + "Weather data unavailable — retrying..." with retry countdown |
| Error state (has prior data) | Preserve last-known data, show amber warning bar with same error copy |

## Interaction & Behavior

| Behavior | Specification |
|----------|---------------|
| Poll interval | 10 minutes (`useWeather` hook matches `useMatchPoller` pattern) |
| Page visibility | Pause polling when tab hidden, immediate refetch on visibility change |
| Retry backoff | Exponential backoff: 1s / 2s / 4s (3 retries), matching `useMatchPoller` |
| Preserve stale data | On subsequent errors, keep last-known data and show amber warning bar |
| Auto re-simulate | When weather condition changes (e.g., clear→rain), re-run `simulateDeterministic()` with adjusted params |

  <action>
    Create `src/hooks/useAlertStream.ts` — a client-side SSE hook modeled on `useMatchPoller` (D-15-10). The hook manages an `EventSource` connection to `/api/alert`, parses incoming SSE events, dispatches alerts to the Zustand alert store, and handles reconnection with exponential backoff.

    ```typescript
    "use client";
    import { useEffect, useRef, useCallback } from "react";
    import { useLiveStore } from "@/stores/liveStore";
    ```

    **SSE Event types** (matching the route handler's output):
    ```typescript
    type SSEEvent =
      | { type: "token"; text: string }
      | { type: "alert"; alert: AlertEvent }
      | { type: "error"; message: string }
      | { type: "complete" };
    ```

    **Connection management:**
    - `const addAlert = useLiveStore((s) => s.addAlert);` — selector for addAlert
    - `const match = useLiveStore((s) => s.match);` — current match state for URL params
    - Use `useRef` for: `EventSource`, retry delay counter, disconnected flag
    - `RECONNECT_BASE = 1_000` (1s), `RECONNECT_MAX = 8_000` (8s) per D-15-12

    **`connect` callback:**
    1. Close existing EventSource if any
    2. Build URL with match state query params: `/api/alert?minute=${match.minute}&phase=${match.phase}&score=${match.score}` (handle null match with defaults)
    3. Create `new EventSource(url)`
    4. `source.onmessage`: Parse `event.data` as JSON → switch on `type`:
       - `"alert"`: Call `addAlert(data.alert)`, reset retry to base, set disconnected = false
       - `"error"`: Log warning with `console.warn("[alert-stream] Server error:", data.message)`, set disconnected = true
       - `"complete"`: Close source, schedule reconnect after 45_000ms for next analysis cycle
    5. `source.onerror`: Close source, set disconnected = true, calculate delay = `Math.min(retryRef.current, RECONNECT_MAX)`, double retry, schedule reconnect with `setTimeout`
    6. Store `sourceRef.current = source`

    **`useEffect`:**
    - Call `connect()` on mount
    - Return cleanup: `sourceRef.current?.close()`, nullify ref
    - Dependencies: `[connect]` — the connect callback has `addAlert` from selector, which is stable

    **Return value:**
    - Return `const isDisconnected = useRef(false)` — a ref that the AlertFeed component can read (or use a state toggle for re-render)

    **Edge cases (D-15-13):**
    - AbortController pattern: The hook DOES NOT create its own AbortController for the EventSource. The `EventSource` abstraction handles its own connection lifecycle. The `useEffect` cleanup (close + null ref) is the proper cleanup mechanism (matches the useMatchPoller pattern of `clearTimers` + cleanup).
    - If the match state changes significantly (different match), the hook should close and reconnect. This is handled because the connect callback will be recreated (match is a dependency of connect).

    Create `tests/hooks/useAlertStream.test.ts`:
    - Test 1: Hook creates EventSource on mount with correct URL
    - Test 2: `onmessage` with `type:"alert"` calls addAlert and resets retry
    - Test 3: `onmessage` with `type:"complete"` closes source and schedules reconnect
    - Test 4: `onerror` triggers exponential backoff reconnect (1s, 2s, 4s, max 8s)
    - Test 5: Unmount cleans up EventSource (close called)
    - Use `vi.useFakeTimers()` for timing control per the useMatchPoller test pattern
    - Mock `EventSource` by replacing `global.EventSource` with a class that exposes `onmessage`, `onerror`, `close`
    - Mock `useLiveStore` to return `addAlert` spy
    - After each test: `vi.restoreAllMocks()`, `vi.useRealTimers()`

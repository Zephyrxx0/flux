## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→API | Fan chat messages cross from browser to server — unvalidated user input enters the prompt |
| API→Gemini | Prompt (including user content) sent to Gemini REST API — injection surface |
| Gemini→API | Structured JSON response received from Gemini — must be validated before delivery to client |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01 | Information Disclosure | `process.env.GEMINI_API_KEY` | mitigate | Server-only route.ts; env var never returned in SSE events. Error events say "API key not configured" without revealing key. |
| T-16-04 | Spoofing | `buildChatPrompt()` prompt assembly | mitigate | System prompt constrains response domain ("Use ONLY the zone data provided"). Retry does not relax constraints. Redirects out-of-scope questions. |
| T-16-02 | Tampering | `parseChatResponse()` Zod validation | mitigate | ChatResponseSchema safeParse() validates every Gemini response. Retry on failure. Text-only fallback if both attempts fail. |
| T-16-SC | Tampering | npm/pip/cargo installs | accept | Phase 16 adds zero new packages. No package-install tasks. |
</threat_model>
